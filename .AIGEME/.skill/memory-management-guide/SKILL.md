# Memory Management Guide
## Memory 工具箱完整使用指南

**10 种 memory 操作**：`add` `read` `search` `list` `del` `link` `audit` `merge` `prune` `graph_search`
**4 种专用编辑工具**：`memory_edit_content` `memory_edit_tags` `memory_edit_title` `memory_edit_importance`

---

# 第一部分：记忆管理流程

## 一、重要度规则（核心）

### 原则：文件级重要度

每份 `.md` 文件只有一个整体的 `importance`，不存在条目级重要度。

| 重要度 | 值 | 适用场景 |
|--------|----|---------|
| ⭐⭐⭐⭐⭐ 核心永久 | 5 | 不可逆的架构决策、持续的用户偏好、必须记住的约定 |
| ⭐⭐⭐⭐ 重要 | 4 | 频繁参考的信息、关键配置、重要工作流 |
| ⭐⭐⭐ 普通（默认） | 3 | 一般事实、中等价值的经验 |
| ⭐⭐ 低价值 | 2 | 临时信息、不再活跃的决策 |
| ⭐ 临时 | 1 | 一次性提示、即将被覆盖的信息 |

### 操作规则

| 操作场景 | 重要度行为 | 说明 |
|---------|-----------|------|
| **新文件** `add(title="新")` | **必传** `importance` | LLM 评估整份文件的整体价值 |
| **追加已有文件** `add(title="已有")` | **忽略** `importance` | 返回"记忆重要度未变更，当前记忆重要度：X" |
| **相似度命中自动追加** | **忽略** `importance` | 同上，返回当前重要度 |
| **显式调整** `memory_edit_importance(title="...", new_importance=Y)` | **更新** frontmatter | 唯一改变重要度的方式 |

### 什么时候调重要度？

- **升级**：文件从零星记录积累成完整知识体系
- **降级**：文件内容已被覆盖/否定，或该主题不再活跃
- **合并后**：合并后的文件分量增加，考虑升一级
- **不要**每次追加都调，重要度是相对稳定的评价

## 二、文件组织策略

### 一份文件的合适粒度

| 内容 | 合适做法 |
|------|---------|
| 一个独立偏好 | 独立文件，如 `用户-作息习惯.md` |
| 一个技术决策 | 独立文件，如 `API设计决策.md` |
| 一个工作流 | 独立文件，如 `部署流程.md` |
| 会话记录 | 按天，如 `会话记录-2026-07-02.md` |
| 多个相关观点 | 同一文件（共享同一重要度） |
| 无关主题混在一起 | **必须拆分**（超过 3 个不直接相关的主题） |

### 什么时候用 `[[wikilink]]`？

两个文件**有关联但不重叠** → 正文中使用 `[[文件名]]` 建立双向链接。
例如"咖啡知识.md"和"过敏记录.md"——有关联但不同主题，不要合并。

### 拆分步骤

```
场景：一份文件 [用户习惯.md] 里混了 作息、饮食、着装 三个无关主题

1. read(title="用户习惯") → 查看全部内容
2. add(title="用户-作息习惯", importance=X, content="...") → 创建新文件
3. add(title="用户-饮食习惯", importance=X, content="...") → 创建新文件
4. 旧文件的条目用 memory_edit_content(title="用户习惯", old_string="...") 逐条删除
5. 删空后用 del(title="用户习惯") 归档
```

### 合并步骤

```
场景：两份文件 [咖啡知识.md] 和 [咖啡技巧.md] 高度重叠

1. read(title="咖啡知识") 和 read(title="咖啡技巧") → 确认重叠度
2. merge(sources=["咖啡知识", "咖啡技巧"], target="咖啡知识汇总")
3. memory_edit_importance(title="咖啡知识汇总", new_importance=5) → 合并后的文件分量增加，升级
```

## 三、定期维护节奏

### 每 15-20 轮对话：`audit`

关注以下 5 项：

| 审计发现 | 含义 | 处理方式 |
|---------|------|---------|
| 断链 | 某文件引用了已删除的文件 | 用 `link` 重新关联或忽略 |
| 孤立文件 | 文件不在任何链接图谱中 | 补充 `[[链接]]` 或 `prune` 清理 |
| 高相似度文件对 | 两文件内容高度重复 | 用 `merge` 合并 |
| 交叉引用缺失 | A 引用了 B，B 没有回链 | 在 B 中补充 `[[A]]` |
| 重要度异常 | 文件内容膨胀但重要度低 | 用 `memory_edit_importance(title="...", new_importance=Y)` 调整 |

### 整理优先级

```
高重要度 > 高引用数 > 大文件
```

# 第二部分：工具使用说明

## `add` — 新增记忆

**参数**：
| 参数 | 必填 | 说明 |
|------|------|------|
| `content` | ✅ | 记忆内容正文 |
| `title` | 可选 | 显示标题。**新标题 = 创建新文件；已有标题 = 追加到已有文件** |
| `importance` | 新文件必填 | 1-5。追加时**不要传**（会被忽略） |
| `type` | 可选 | `fact`(默认) / `preference` / `task_status` / `event` / `emotion` / `reflection` / `process` |
| `tags` | 可选 | 标签列表，如 `["项目", "经验"]` |

**执行顺序**：
1. **相似度查重** — 与已有内容相似度 ≥ 0.7，自动合并到已有文件
2. **title 路由** — 未命中相似度时，同 title 追加到同一文件
3. **新建文件** — 都不命中时，创建新 `.md` 文件

**追加时的返回值**：
```
记忆重要度未变更，当前记忆重要度：3
```

**在正文中使用 `[[文件名]]` 建立链接**，系统自动更新双向链接图谱。

---

## `read` — 读取记忆

**参数**：
| 参数 | 说明 |
|------|------|
| `title` | 按标题查找（二选一） |
| `id` | 按 ID 精确查找（二选一，优先于 title） |

返回文件完整内容（frontmatter + 正文）。

---

## `search` — 搜索关键词

**参数**：
| 参数 | 必填 | 说明 |
|------|------|------|
| `query` | ✅ | 搜索关键词（内容 + 文件名） |
| `tags_filter` | 可选 | 只返回包含指定标签的结果 |
| `title` | 可选 | 限定特定文件内搜索 |

使用倒排索引 + jieba 分词，搜索范围为文件正文**和 tags 标签值**。结果按相关性排序，超过 30 天未引用的文件会被降权。

---

## `list` — 浏览索引

**参数**：
| 参数 | 说明 |
|------|------|
| `include_all` | 设为 `true` 时包含过期文件（30 天未引用） |
| `type` | 按类型筛选（可选） |

返回 MEMORY.md 索引表，按类型分区显示。

---

## `del` — 删除记忆

**参数**：
| 参数 | 说明 |
|------|------|
| `title` | 按标题删除（二选一） |
| `id` | 按 ID 删除（二选一，优先于 title） |

> 软删除 — 文件移入 `_archive/` 目录，物理文件不丢失。核心文件（MEMORY.md、LINKS.md）不可删除。

---

## `memory_edit_content` — 编辑记忆正文

**参数**（`title` 和 `old_string` 必填）：

| 参数 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 要编辑的记忆标题 |
| `old_string` | ✅ | 要替换的原文（必须唯一匹配） |
| `new_string` | 可选 | 替换后的新内容，默认空字符串（即删除原文） |

**注意**：`old_string` 在文件中只能出现一次，否则报错。建议先用 `read` 查看全文，复制要替换的精确原文。正文修改不会影响 frontmatter 元数据（tags/importance/title 等）。

---

## `memory_edit_tags` — 编辑记忆标签

**参数**（`title` 和 `tags` 必填）：

| 参数 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 要编辑的记忆标题 |
| `tags` | ✅ | 新的标签列表（完全覆盖现有标签） |

**示例**：`memory_edit_tags(title="出差记录", tags=["工作", "差旅"])`

---

## `memory_edit_title` — 编辑记忆显示标题

**参数**（`title` 和 `new_title` 必填）：

| 参数 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 当前的记忆标题 |
| `new_title` | ✅ | 新的显示标题（仅修改 frontmatter，不改变文件名） |

**注意**：`title` 是查找依据，`new_title` 是更改后的值。例如 `memory_edit_title(title="出差记", new_title="出差记录")` 修正错别字。

---

## `memory_edit_importance` — 编辑记忆重要度

**参数**（`title` 和 `new_importance` 必填）：

| 参数 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 要编辑的记忆标题 |
| `new_importance` | ✅ | 新的重要度 1-5（1=临时 / 2=低价值 / 3=普通 / 4=重要 / 5=核心永久） |

---

## `link` — 显式建立链接

**参数**：
| 参数 | 说明 |
|------|------|
| `src` | 链接源文件名（不含 .md） |
| `tgt` | 链接目标文件名（不含 .md） |

**什么时候需要显式 link**？
- 两个文件正文中没有互相引用，但需要在图谱中关联
- 更推荐的做法：在正文中直接用 `[[文件名]]` 语法，系统自动处理

两种方式都写入 LINKS.md，对检索透明。

---

## `audit` — 扫描审计

无需额外参数。返回：
- 断链列表（links 指向了不存在的文件）
- 孤立文件（目录中存在但不在任何链接图中）
- 高相似度文件对（内容高度重复）
- 交叉引用缺失（A 引用了 B，B 没有回链）

审计后参考第一部分的维护策略处理。

---

## `merge` — 合并文件

**参数**：
| 参数 | 说明 |
|------|------|
| `sources` | 待合并的源文件列表（必填） |
| `target` | 合并目标文件名（必填，可以是新名称或已有名称） |

**重要度处理**：合并后需要手动 `memory_edit_importance(title="...", new_importance=Y)` 调整目标文件的重要度。

---

## `prune` — 清理孤立文件

无需参数。自动将不在任何链接图谱中的孤立文件移到 `_archive/`。
执行前建议先 `audit` 确认哪些文件是孤立的。

---

## `graph_search` — 图谱扩散检索

**参数**：
| 参数 | 必填 | 说明 |
|------|------|------|
| `seed` | ✅ | 起始节点关键词（不要求精确匹配文件名，系统会匹配后 BFS 扩散） |
| `max_depth` | 可选 | 扩散层数，默认 2（depth=1 权重 0.8，depth=2 权重 0.6） |
| `query_tags` | 可选 | 只返回含指定标签的节点 |

**原理**：
1. 从 seed 节点出发，通过 LINKS.md 的双向链接关系做 BFS 扩散
2. 信号融合：图谱深度 + 标签匹配 + 时间衰减
3. 匹配流程：**精确节点名 → 模糊节点名（seed 包含节点名或反之）→ 正文搜索（含 tags）**

**method 返回值说明**：
| method | 含义 |
|--------|------|
| `graph_search` | 精确节点名命中，BFS 扩散 |
| `graph_search_fuzzy` | 模糊节点名命中（seed 包含节点名或反之），BFS 扩散 |
| `content_fallback` | 未找到匹配节点，降级为正文关键词搜索（含 tags） |

**什么时候手动调用**？
- 自动检索（每轮对话的 memory_tracker）使用 jieba 分词提取 seed
- 分词器可能漏掉语义同义词（如"AI"和"人工智能"）
- LLM 手动调用 `graph_search(seed="AI")` 可以绕过分词器局限

---

## 附录：`[[wikilink]]` 自动链接

在新增记忆的正文中使用 `[[文件名不含后缀]]` 语法：
- 系统自动解析所有 `[[...]]`，更新 LINKS.md
- 建立双向链接关系
- 支持 `audit` 的断链检测和交叉引用完整性
